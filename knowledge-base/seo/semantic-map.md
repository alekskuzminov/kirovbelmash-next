## Принцип

Каждый запрос принадлежит ровно одному URL. Это предотвращает каннибализацию — когда несколько страниц конкурируют за одни и те же позиции.

## Правило проверки нового текста

Перед добавлением запроса в текст страницы, проверь:
1. Определи интент запроса — пользователь хочет купить конкретную линию или узнать о производителе?
2. Найди запрос в карте выше — к какому URL он принадлежит?
3. Если пишешь текст для этой страницы — запрос можно использовать. Если нет — **не добавляй**.

## Опасные формулировки для главной страницы

На главной **нельзя** системно выносить в Title, H1, SEO-текст:

- ~~купить линию брикетирования~~
- ~~купить линию гранулирования~~
- ~~оборудование для производства брикетов~~
- ~~оборудование для производства пеллет~~
- ~~линия по производству брикетов~~
- ~~линия по производству пеллет~~

**Можно** на главной (широкий контекст):
- «производитель промышленного оборудования и линий для переработки древесных отходов»
- «в составе решений: линии брикетирования, линии гранулирования и сушильное оборудование»

## Чеклист SEO для новых страниц

- [ ] Title: до 54 символов, главный запрос кластера в начале
- [ ] H1: отличается от Title, содержит основной + второй кластер
- [ ] Description: до 160 символов, с ценой и производительностью
- [ ] H2 секции: каждый покрывает отдельный поисковый кластер
- [ ] JSON-LD: BreadcrumbList + ItemList (для страниц линий) + FAQPage
- [ ] SEO-текстовый блок: двухколоночный layout, quote, список состава, ссылки на услуги
- [ ] FAQ: 5 вопросов под конкретные запросы с просадкой
- [ ] Цена: в Description, контенте (подзаголовок), FAQ, JSON-LD
- [ ] Внутренние ссылки: минимум 4 на услуги + смежные линии
- [ ] Нет каннибализации: запросы кластера не дублируются на другой URL
