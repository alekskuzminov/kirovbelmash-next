# База знаний КировБелМаш

Проверенная, актуальная информация для генерации SEO-текстов, статей блога и описаний оборудования.

## Принцип работы

- **Только верифицированные факты** используются в публичном контенте. Флаг `verified: true` в frontmatter означает, что заказчик проверил информацию.
- **Числа, модели, цены** — строго из этих файлов. Ничего не выдумываем.
- **Пополнение** — через команду `/kb-add` или вручную по шаблону ниже.

## Структура

```
knowledge-base/
├── company/          — О компании: история, преимущества, производство
├── products/         — Технические данные по направлениям (брикетирование, гранулирование и т.д.)
├── industry/         — Отраслевые знания: терминология, сырьё, нормативы, FAQ
└── seo/              — Семантика, ключевые слова, карта запросов
```

## Шаблон нового файла

```markdown
---
name: Название темы
description: Одна строка — что здесь хранится
type: product | company | industry | seo
verified: false
verified_by:
verified_date:
updated: YYYY-MM-DD
tags: [тег1, тег2]
---

## Раздел

Контент...
```

## Статусы верификации

| Статус | Значение |
|--------|---------|
| `verified: true` | Проверено заказчиком, можно использовать в публичных текстах |
| `verified: false` | Черновик, требует проверки |
| `verified: partial` | Часть данных проверена (указаны конкретные разделы в файле) |

## Как пополнять

1. **Через скилл** (рекомендуется): `/kb-add` — передаёшь сырой текст, Claude структурирует и кладёт в нужный файл
2. **Вручную**: создай файл по шаблону выше, добавь запись в `_index.md`
3. **После проверки заказчиком**: обновить `verified: true`, `verified_by`, `verified_date`
