# SEO — рабочая директория

Здесь хранятся рабочие артефакты SEO-работы с подрядчиком.  
**Не путать с `knowledge-base/seo/`** — туда переносятся только верифицированные данные, готовые к использованию в контенте.

---

## Структура

```
seo/
├── audit/
│   ├── technical/    # Технический аудит: скорость, краулинг, индексация, robots, sitemap
│   └── content/      # Контентный аудит: страницы, gaps, дубли, thin content
├── research/
│   ├── keywords/     # Семантика от подрядчика (CSV, кластеры, mind-maps)
│   ├── competitors/  # Анализ конкурентов
│   └── serp/         # Анализ выдачи по приоритетным запросам
├── tasks/
│   └── backlog.md    # Бэклог задач
├── reports/
│   └── YYYY-MM.md    # Ежемесячные отчёты
└── content-plan/
    └── blog-plan.md  # Контент-план блога
```

---

## Поток данных

```
Подрядчик → seo/research/ или seo/tasks/backlog.md
                    ↓
            Верификация владельцем
                    ↓
        knowledge-base/seo/ (верифицированные данные)
                    ↓
     Claude генерирует контент на основе KB (/blog-article, /kb-add)
                    ↓
          blogData.ts / страницы сайта
```

---

## Правила работы

1. **Сырые данные** от подрядчика — в `research/`. Не редактировать, хранить как получили.
2. **Задачи** — в `tasks/backlog.md`. Статусы: 🔴 не начата / 🟡 в работе / 🟢 выполнена.
3. **Отчёты** — один файл на месяц: `reports/YYYY-MM.md`.
4. **В knowledge-base/seo/** переносить только после верификации владельцем.
5. **Числа и факты** — не выдумывать. Если данных нет — ставить `TODO:`.
