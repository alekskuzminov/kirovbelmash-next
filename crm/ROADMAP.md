# CRM Roadmap

Живой документ. По мере выполнения задач — двигать пункты между секциями. После завершения фазы — фиксировать дату.

## Фаза 1 — Фундамент ✅ (2026-04-27)

- [x] PostgreSQL на VPS
- [x] Prisma-схема: User, Contact, Pipeline, Stage, Deal, Task, Note, Document
- [x] NextAuth (Credentials, JWT)
- [x] Защита `/admin/*` через `src/proxy.ts`
- [x] Layout: тёмный sidebar, страница входа
- [x] Seed: admin-пользователь + дефолтная воронка
- [x] Деплой на продакшен

## Фаза 2 — Ядро CRM (в работе)

### Сделки
- [ ] Server Action / API для CRUD сделок
- [ ] Kanban-доска (drag-n-drop между этапами)
- [ ] Карточка сделки: контакт, этап, ответственный, сумма, источник, заметки
- [ ] Создание сделки вручную из UI
- [ ] Фильтры (по этапу, ответственному, дате)

### Контакты
- [ ] Список контактов (таблица + поиск)
- [ ] Карточка контакта: реквизиты, история сделок
- [ ] CRUD контакта

### Интеграция с сайтом
- [ ] В `/api/contact/route.ts` — после `sendMail` создавать контакт + сделку в БД
- [ ] Дедупликация по телефону (если контакт с таким номером есть — добавить сделку к нему)
- [ ] Статус «source» в сделке = откуда заявка (форма какой страницы)
- [ ] Если БД упала — заявка всё равно уходит на почту (письмо приоритетнее)
