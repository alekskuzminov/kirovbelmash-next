## Фаза 3 — Карточка сделки и документы ✅ (2026-04-28)

### Улучшение карточки сделки (DealModal)
- [x] Контакт → кликабельная ссылка на карточку + телефон с `tel:` + email с `mailto:`
- [x] Лента активности: хронологически перемешаны `DealStageEvent` и `Note`
- [x] Бейдж «N дней в этапе» — вычисляется из `DealStageEvent`
- [x] Поле «Город» (`city String?`) на Deal
- [x] Плашка «Нет задач — рекомендуем добавить» в ленте активности
- [x] Даты создания и обновления (`createdAt`, `updatedAt`) в карточке

### Документы к сделке
- [x] Поле `documentUrl String?` на Deal
- [x] API endpoint `POST /api/crm/upload-deal-doc` — загрузка PDF в S3, путь `crm/deals/{dealId}/doc.pdf`
- [x] Кнопка загрузки/замены в карточке сделки
- [x] `PdfPreviewModal` — `<iframe>` поверх карточки, нативный рендер браузера
- [x] Удаление документа (с подтверждением) — очищает S3 + БД

### Kanban-доска
- [x] Дата создания «Создано: DD.MM.YYYY» в правом верхнем углу карточки
- [x] Источник по умолчанию: «Вручную» для ручных сделок, «Сайт» для автоматических
- [x] Три зоны сброса при перетаскивании: «Удалить», «Не реализована», «Успешно завершена»
- [x] Enum `DealResult` (WON/LOST) + поля `closedAs`, `closedAt` на Deal
- [x] Закрытые сделки скрыты из канбана (`closedAt: null` в фильтре)

### Создание сделок — унификация двух потоков
- [x] Автоматический (форма сайта): findOrCreate Contact по phoneNormalized → create Deal
- [x] Ручной: режим «+ Новый контакт» inline в CreateDealModal (имя + телефон + компания)
- [x] Контакт в CreateDealModal: autocomplete-dropdown вместо статичного списка
- [x] Название автосделки: `message` → значения `extra` → «Входящая заявка» (60 символов макс.)

### Задачи
- [ ] CRUD задач
- [ ] Дедлайн, ответственный, привязка к сделке/контакту
- [ ] Виджет «мои задачи на сегодня» в сайдбаре или дашборде
- [ ] Просроченные задачи — подсветка
