Добавить новую информацию в базу знаний или обновить существующую.

## Аргумент
Описание что добавить и источник. Примеры:
- `/kb-add технические характеристики ОГМ-1.5 из письма заказчика`
- `/kb-add новые данные по ценам линий`
- `/kb-add кейс — завод в Белгороде, 800 кг/ч`
- `/kb-add обновить статус verified для products/granulirovanie.md`

Источник данных может быть передан:
- как текст прямо в чате
- как путь к файлу (PDF, текст, заметка)

## Алгоритм выполнения

### Шаг 1. Прочитай индекс
Прочитай `knowledge-base/_index.md` — понять, какие файлы уже есть и куда может лечь новая информация.

### Шаг 2. Определи тип информации
Классифицируй входящую информацию:

| Что за информация | Куда класть |
|------------------|------------|
| О компании, производстве, команде | `company/` |
| Технические данные оборудования, состав линий | `products/` |
| Отраслевые знания, сырьё, термины, FAQ | `industry/` |
| Ключевые слова, запросы, карта URL | `seo/` |
| Кейс клиента, реализованный проект | `cases/` (создать если нет) |

### Шаг 3. Прочитай целевой файл
Прочитай файл, в который будешь добавлять данные. Если файла нет — создашь новый по шаблону.

### Шаг 4. Внеси изменения
Правила:
- **Факты и числа** — добавлять точно, без округления. Если есть диапазон — указывать диапазон.
- **Непроверенные данные** — добавлять с пометкой `<!-- TODO: уточнить у заказчика -->` или в раздел с явным заголовком «Требует уточнения»
- **Устаревшие данные** — обновлять, не удалять старые значения сразу (оставить как `<!-- Было: ... -->`)
- **Дублирующиеся данные** — объединять, не плодить дубликаты
- **Обновить поле `updated`** в frontmatter на дату сегодня

### Шаг 5. Если данные проверены заказчиком
Обновить в frontmatter:
```yaml
verified: true
verified_by: [имя/должность]
verified_date: YYYY-MM-DD
```

### Шаг 6. Обновить `_index.md`
Если создан новый файл — добавить запись в `knowledge-base/_index.md`.

### Шаг 7. Сообщи итог

Сообщи:
- В какой файл добавлено
- Какие поля обновлены
- Что помечено как `TODO` (требует уточнения)
- Если были конфликты с существующими данными — указать явно

## Шаблон нового файла

```markdown
---
name: Название
description: Одна строка — что хранится
type: product | company | industry | seo | case
verified: false
verified_by:
verified_date:
updated: YYYY-MM-DD
tags: [тег1, тег2]
---

## Раздел

Контент...
```
