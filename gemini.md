# Правила работы с проектом (Antigravity AI Rules)

Этот файл содержит основные инструкции и ограничения для ИИ-ассистента при работе с проектом.

## 0. ЯЗЫК ОБЩЕНИЯ И КОД (КРИТИЧЕСКИ ВАЖНО)
- **Общение в чате:** Все ответы, рассуждения, планы и отчеты ИИ обязан писать СТРОГО НА РУССКОМ ЯЗЫКЕ.
- **Написание кода:** Весь исходный код, имена переменных, названия файлов и папок — СТРОГО НА АНГЛИЙСКОМ.
- **История Git:** Сообщения коммитов (`git commit -m`) писать исключительно на английском языке по общепринятым стандартам (например, `feat: add hero section`).
- **Запрет на кириллицу:** Категорически запрещено использовать кириллицу и пробелы в названиях файлов (особенно в папке `public/images/`). Использовать только латиницу, `kebab-case` или `snake_case` (например, `drying_lines.jpg`).

## 1. Технологический стек
- **Фреймворк:** Next.js 16 (App Router, Turbopack). Роутинг через файловую систему (`src/app/`).
- **Язык:** TypeScript. Использовать строгую типизацию, избегать `any`.
- **Стилизация:** Tailwind CSS 4. Использовать utility-классы Tailwind. Кастомный CSS писать в `globals.css` только в крайнем случае.
- **UI Библиотеки:** Сторонние библиотеки компонентов (MUI, AntD, Shadcn) не используются. Все компоненты пишутся с нуля.
- **Дополнительно:** `recharts` для графиков (при необходимости).

## 2. Архитектура и структура папок
- `src/app/` — страницы и layouts (App Router). Каждая папка = маршрут.
- `src/app/layout.tsx` — корневой layout (Navbar, Footer, шрифты, мета-теги).
- `src/app/page.tsx` — главная страница.
- `src/app/[route]/page.tsx` — страницы по маршрутам.
- `src/components/` — общие, переиспользуемые UI-компоненты (Navbar, Footer, кнопки и т.д.).
- `src/lib/` — утилиты и вспомогательные функции.
- `src/types/` — общие TypeScript-типы и интерфейсы.
- `src/data/` — статические данные (замена `mocks/` для SSG-контента).
- `public/images/` — статические ассеты (изображения).

**Главное правило:** Не создавать новые корневые папки без явного указания. Соблюдать конвенции Next.js App Router.

## 3. Next.js — специфичные правила
- **Server Components по умолчанию.** Все компоненты — серверные, если не указано иное.
- **`"use client"`** — добавлять ТОЛЬКО когда компонент использует: `useState`, `useEffect`, `useRef`, `onClick`, `onChange` и другие браузерные API/хуки.
- **Навигация:** Использовать `<Link>` из `next/link` вместо `<a>` для внутренних ссылок.
- **Изображения:** Использовать `<Image>` из `next/image` вместо `<img>`. Указывать `width`, `height` или `fill`. Для изображений из `public/` путь начинается с `/` (например, `/images/logo.webp`).
- **Метаданные:** Использовать `export const metadata: Metadata = {}` в `layout.tsx` и `page.tsx` (серверные компоненты) для SEO.
- **Шрифты:** Подключать через `next/font/google` или `next/font/local` в корневом layout.
- **Запрет `useRouter` для навигации:** Для программной навигации использовать `useRouter` из `next/navigation` (не из `next/router`).

## 4. Команды и скрипты (npm)
- `npm run dev` — запуск сервера разработки (Turbopack).
- `npm run build` — сборка проекта для продакшена.
- `npm run start` — запуск production-сервера.
- `npm run lint` — проверка линтером.

**Правило:** Перед коммитом или по завершении крупной задачи запускать `npm run build` для проверки сборки.

## 5. Бэкенд и Инфраструктура (Целевая архитектура)
- **Серверная часть:** Next.js API Routes (`src/app/api/`) + Node.js.
- **База данных:** PostgreSQL (для хранения структурированных данных, текстов статей, каталогов).
- **Хранилище медиа:** S3-совместимое хранилище (Timeweb Cloud) для изображений и документов.
- *Примечание:* Сторонние BaaS-сервисы (Supabase, Firebase, Notion API) в проекте НЕ используются. До подключения реального бэкенда работа идет со статическими данными в `src/data/`.

## 6. Правила написания кода (Code Style & Taboos)
- **Функциональные компоненты:** Использовать только стрелочные функции для создания компонентов (`const MyComponent = () => {}`).
- **Комментарии:** Не удалять существующие комментарии разработчика. Добавлять комментарии только к сложной бизнес-логике.
- **Конфиги:** Запрещено изменять конфигурационные файлы (`next.config.ts`, `tailwind.config.ts`, `package.json`, `eslint.config.mjs`, `tsconfig.json`) без прямого указания пользователя.
- **Импорты:** Использовать алиас `@/` для абсолютных импортов (например, `import { Button } from '@/components/Button'`). Поддерживать порядок импортов (внешние библиотеки → внутренние модули → стили).
- **Состояние:** Использовать хуки React (`useState`, `useEffect`, `useContext`) только в клиентских компонентах (`"use client"`).

## 7. Безопасность и хранение данных (Credentials)
- **СТРОГИЙ ЗАПРЕТ:** Никогда не вписывай логины, пароли, API-ключи напрямую в исходный код.
- Все секретные ключи должны храниться исключительно в файле `.env.local` и вызываться через `process.env.VARIABLE_NAME`.
- Публичные переменные (доступные в браузере) должны иметь префикс `NEXT_PUBLIC_`.

## 8. Миграция из старого проекта
- **Источник:** Старый проект `kirovbelmash` (React + Vite) доступен в рабочей области.
- **Правила переноса:**
  - Заменять `react-router-dom` (`<Link to="">`, `useNavigate`) на `next/link` и `next/navigation`.
  - Заменять `<img>` на `<Image>` из `next/image`.
  - Заменять `<a href="">` на `<Link href="">` для внутренних ссылок.
  - Добавлять `"use client"` только к интерактивным компонентам.
  - Переносить стили Tailwind без изменений (максимальная совместимость).
  - Изображения из `public/images/` копировать как есть.

## 9. Оркестрация, MCP и обработка ошибок
- **Режим планирования:** Перед изменением существующего кода всегда формируй пошаговый план действий и запрашивай подтверждение.
- **Актуализация знаний:** При написании сложного кода для Next.js 16 или Tailwind CSS 4 приоритетно обращайся к MCP-серверу Context7 за актуальной документацией.
- **Ошибки в терминале:** При возникновении ошибки сборки не пытайся исправить её слепым перебором. Остановись, проанализируй логи и предложи осмысленное решение.
