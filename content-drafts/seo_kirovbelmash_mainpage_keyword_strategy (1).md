# Инструкция: как продвигать главную страницу kirovbelmash.ru без пересечения с разделами линий

**Сайт:** `https://kirovbelmash.ru/`  
**Связанные посадочные:**  
- `https://kirovbelmash.ru/linii-briketirovaniya`
- `https://kirovbelmash.ru/linii-granulirovaniya`

**Дата подготовки:** 29.03.2026

---

## 1. Задача

Нужно продвигать главную страницу так, чтобы она:

- собирала широкий коммерческий и брендовый спрос;
- не конкурировала в поиске с двумя продуктовыми страницами;
- не размывала релевантность разделов по брикетированию и гранулированию;
- работала как корпоративная и обзорная точка входа в сайт.

Проще говоря:

- **главная** отвечает на вопрос: **«кто вы, что производите в целом, какие задачи закрываете»**;
- **страница линий брикетирования** отвечает на вопрос: **«нужно оборудование / линия для производства брикетов»**;
- **страница линий гранулирования** отвечает на вопрос: **«нужно оборудование / линия для производства пеллет»**.

---

## 2. Базовый принцип разведения запросов

Чтобы не было SEO-каннибализации, у каждой страницы должен быть свой доминирующий интент.

### Главная страница
Закрепляем за ней:

- бренд;
- производитель / завод;
- комплексные решения;
- переработка древесных отходов в целом;
- запуск производства «под ключ»;
- промышленное оборудование широкого профиля без ухода в конкретный продукт.

### Страница `/linii-briketirovaniya`
Закрепляем за ней всё, что связано с:

- брикетированием;
- топливными брикетами;
- RUF / Pini Kay;
- линиями и оборудованием для производства брикетов;
- покупкой линии брикетирования.

### Страница `/linii-granulirovaniya`
Закрепляем за ней всё, что связано с:

- гранулированием;
- пеллетами;
- пеллетным оборудованием;
- линиями для производства пеллет;
- покупкой линии гранулирования.

Это означает, что **главная уже частично пересекается с продуктовыми посадочными**.

## 7. Что нужно скорректировать на главной, чтобы не было пересечения

Сейчас главная уже содержит сильные продуктовые сигналы. Это не критично, но для чистого разведения лучше ослабить прямые продуктовые формулировки.

### 7.1. Что желательно ослабить

Нежелательно делать главным SEO-акцентом на главной:

- «линии брикетирования и гранулирования» как основной ключ;
- «производство брикетов и пеллет» как доминирующий тезис;
- «оборудование для производства брикетов / пеллет» в повторяющемся виде.

### 7.2. Во что это лучше переформулировать

Вместо слишком продуктовых конструкций использовать более широкие:

- оборудование для переработки древесных отходов;
- производственные линии полного цикла;
- производитель промышленного оборудования;
- комплексные решения под ключ;
- проектирование, производство, запуск и сервис;
- линии для переработки опилок, щепы и древесных отходов.

### 7.3. Рекомендуемый смысловой каркас главной

Главная должна транслировать:

1. Кто вы — завод / производитель.
2. Что делаете — проектируете и производите промышленные линии.
3. Для чего — переработка древесных отходов.
4. В каком формате — под ключ.
5. Какие направления есть — брикетирование, гранулирование, сушка.
6. Куда вести дальше — в конкретные продуктовые разделы.

То есть главная — это **обзорный хаб**, а не отдельная страница под запрос «купить линию гранулирования».

### 8.1. Каким должен быть Title главной

Title должен быть широким и корпоративно-коммерческим.

Подходящие варианты:

### 8.2. Каким должен быть H1 главной

H1 также лучше делать широким.

Подходящие варианты:

- Производство оборудования для переработки древесных отходов
- Производственные линии полного цикла для переработки древесных отходов
- Завод-производитель оборудования и линий переработки древесных отходов

### 8.3. Чего избегать в Title/H1 главной

Избегать как основных формулировок:

- оборудование для производства пеллет из опилок;
- оборудование для производства топливных брикетов;
- купить линию гранулирования;

Эти формулировки должны работать на внутренних страницах.

## 9. Как писать текст на главной без каннибализации

### Можно

- упоминать, что компания производит линии брикетирования, гранулирования и сушки;
- кратко перечислять направления;
- давать анонсы категорий и ссылки на них;
- описывать производственные возможности, сервис, запуск, гарантию, проекты;
- раскрывать компетенцию завода, а не конкретный продуктовый спрос.

### Нельзя

- делать длинные SEO-блоки под фразы «оборудование для производства пеллет» и «оборудование для производства брикетов» одновременно;
- повторять точные продуктовые ключи по 5–10 раз в теле главной;
- вставлять на главную коммерческие подзаголовки, которые по смыслу должны жить в продуктовых разделах;
- пытаться продвигать одну страницу сразу и под бренд, и под все товарные кластеры.

### Правильная логика текста главной

Не:

> Мы производим оборудование для брикетов, оборудование для пеллет, линии брикетирования, линии гранулирования, станки для брикетов, грануляторы для пеллет...

А:

> КировБелМаш — производитель промышленного оборудования и линий полного цикла для переработки древесных отходов. В составе решений: линии брикетирования, линии гранулирования и сушильное оборудование.

- главная — общий центр тематики;

## 11. Практическая схема работы с семантикой

### Шаг 1. Собрать все запросы

Соберите общее ядро по темам:

- производитель оборудования;
- переработка древесных отходов;
- линии под ключ;
- брикетирование;
- гранулирование;
- пеллеты;
- брикеты;
- сушильные линии;
- оборудование для опилок и щепы.

### Шаг 2. Разметить интент каждого запроса

Каждый запрос помечайте как один из типов:

- **брендовый**;
- **общий отраслевой**;
- **коммерческий широкий**;
- **продуктовый брикеты**;
- **продуктовый пеллеты**;
- **информационный**.

### Шаг 3. Распределить по URL

Принцип:

- брендовые и широкие коммерческие → на главную;
- брикетные → на `/linii-briketirovaniya`;
- пеллетные → на `/linii-granulirovaniya`;
- информационные → в блог / статьи.

### Шаг 4. Проверить конфликтные запросы

Если запрос содержит:

- «брикет» / «брикетирование» / `ruf` / `pini kay` → почти всегда это страница брикетирования;
- «пеллет» / «гранулирование» / «гранулятор» / «пеллетайзер» → почти всегда это страница гранулирования.

Если запрос содержит только широкий смысл:

- производитель;
- завод;
- переработка древесных отходов;
- линия под ключ;
- оборудование полного цикла;

то это кандидат на главную.

### Шаг 5. Зафиксировать запрет на пересечение

После распределения составьте таблицу:

- запрос;
- целевой URL;
- запрещённые URL.

Это сильно упрощает дальнейшую SEO-работу и правки контента.

## 12. Как контролировать, что пересечения нет

### Проверка 1. По логике страницы

Задайте вопрос:

**Если пользователь вводит запрос, он хочет посмотреть компанию в целом или конкретно купить определённую линию?**

- если компанию / производителя / варианты решений → главная;
- если конкретную линию → внутренняя страница.

### Проверка 2. По Title / H1

Если на двух URL одинаковые или очень близкие Title/H1, это тревожный сигнал.

### Проверка 3. По выдаче

Проверяйте, какой URL чаще показывается по кластеру:

- если по запросу «линия гранулирования» стабильно показывается главная — распределение сделано плохо;
- если по запросу «производитель оборудования для переработки древесных отходов» показывается страница гранулирования — тоже плохо.

### Проверка 4. По Google Search Console / Яндекс Вебмастеру

Нужно смотреть:

- по каким запросам показывается каждая страница;
- нет ли наложения одних и тех же кластеров на несколько URL;
- не проседают ли показы продуктовых страниц после усиления главной.

## 15. Короткий вывод

Для главной страницы `kirovbelmash.ru` нужно продвигать **не товарные запросы про брикеты и пеллеты**, а **широкую коммерческую семантику производителя оборудования для переработки древесных отходов**.

Главная должна отвечать на интент:

- производитель;
- завод;
- комплексные решения;
- запуск под ключ;
- переработка древесных отходов;
- несколько направлений производства в рамках одного поставщика.

А всё, что связано с:

- **линиями брикетирования**,
- **оборудованием для производства брикетов**,
- **линиями гранулирования**,
- **оборудованием для производства пеллет**,

нужно закреплять за соответствующими внутренними страницами.

Именно это даст наиболее чистую структуру без каннибализации и с понятным распределением релевантности между URL.
