# SEO-оптимизация страниц линий — Playbook

Инструкция по SEO-оптимизации страниц категорий линий (`/linii-granulirovaniya`, `/linii-briketirovaniya`, `/sushilnie-linii`).

Разработана на основе оптимизации страницы `/linii-granulirovaniya` (март 2026).

---

## Предварительный этап: сбор семантики

Перед началом работы над страницей необходимо:

1. **Собрать поисковые запросы** из Яндекс.Вебмастера / Метрики по URL страницы
2. **Отсортировать по частотности** — определить ядро (топ-5 запросов по частоте)
3. **Зафиксировать текущие позиции** — чтобы отслеживать динамику после изменений
4. **Определить просадки** — запросы с высокой частотой и позицией хуже 10

---

## Шаг 1. Title

**Формула:** `[Главный запрос по частоте] — [второй кластер]`

**Ограничения:**
- Основная часть — до **54 символов** (суффикс `| КировБелМаш` добавляется автоматически через template в `layout.tsx`)
- Итого в выдаче — до **69 символов**
- Самый частотный запрос — **в начале** title
- Второй кластер — через тире

**Пример (гранулирование):**
```
Было:  "Оборудование для производства топливных пеллет"
Стало: "Оборудование для производства пеллет — пеллетные линии"
```

**Антипаттерн:** не выносить в начало title низкочастотные запросы ради «красоты». Частотность = приоритет.

---

## Шаг 2. H1

**Формула:** `headingMain` = главный запрос + уточнение | `headingAccent` = второй кластер

**Правила:**
- H1 должен отличаться от title (не дублировать дословно)
- В `headingMain` — основной запрос + конкретика (сырьё, тип продукции)
- В `headingAccent` (красная строка) — второй кластер + коммерческий триггер

**Пример:**
```
headingMain:   "Оборудование для производства пеллет из опилок"
headingAccent: "линии гранулирования «под ключ»"
```

---

## Шаг 3. Description

**Формула:** `[Основной запрос расширенный] + [цена от X] + [производительность] + [услуги]`

**Правила:**
- До **160 символов**
- Обязательно: цена (от X млн ₽), диапазон производительности
- Включить запросы, которые не вошли в title/H1
- Цена повышает CTR и покрывает запросы типа «... купить цена»

**Пример:**
```
"Оборудование для производства топливных пеллет из опилок и древесных отходов.
Пеллетные линии гранулирования от 9,2 млн ₽. Производительность 500–2000 кг/ч.
Изготовление, монтаж, запуск."
```

---

## Шаг 4. H2 секции

Каждый H2 на странице — возможность покрыть отдельный поисковый кластер.

**Правило:** берём запросы с частотой 15+ которые не вошли в title/H1, и формулируем H2 под них.

**Пример распределения:**
| H2 секция | Целевой запрос |
|-----------|---------------|
| Секция карточек линий | «пеллетное оборудование» (15) |
| Секция процесса | «установка для производства пеллет» (22) |
| SEO-текстовый блок | «купить оборудование ... от производителя» |
| FAQ | «частые вопросы о производстве пеллет» |

## Шаг 5. JSON-LD разметка

На каждой странице линий добавляем **3 схемы**:

### 5.1. BreadcrumbList
```json
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    { "@type": "ListItem", "position": 1, "name": "Главная", "item": "https://kirovbelmash.ru/" },
    { "@type": "ListItem", "position": 2, "name": "Название категории", "item": "https://kirovbelmash.ru/slug" }
  ]
}
```

### 5.2. ItemList с Product
Для каждой линии в категории — `ListItem` с вложенным `Product`:
- `name` — полное название линии с производительностью
- `description` — из `linesData.ts`
- `brand` — КировБелМаш
- `offers.price` — числовое значение из `linesData.ts`
- `offers.priceCurrency` — RUB
- `offers.availability` — InStock
- `offers.priceValidUntil` — конец текущего года

### 5.3. FAQPage
Генерируется автоматически компонентом FAQ (см. шаг 7).

**Размещение:** JSON-LD для BreadcrumbList и ItemList — в `page.tsx` как константы, рендерятся через `<script type="application/ld+json">`. FAQPage — внутри компонента FAQ.

## Шаг 6. SEO-текстовый блок

**Компонент:** `{LineType}SeoText.tsx` в `src/components/lines/`

**Структура (двухколоночный layout):**

```
┌─────────────────────────────────┬──────────────────┐
│  H2 с красной чертой            │                  │
│                                 │  CTA-карточка    │
│  Вводный абзац                  │  (sticky,        │
│                                 │   тёмный фон)    │
│  Quote-блок (красный фон)       │                  │
│  — тезис о линиях               │  • Запросить КП  │
│                                 │  • Телефон       │
│  H3 + список с иконками         │                  │
│  (состав линии, 2 колонки)      │                  │
│                                 │                  │
│  Абзац с внутренними ссылками   │                  │
│  на страницы услуг              │                  │
└─────────────────────────────────┴──────────────────┘
```

**Визуальные элементы (из дизайн-системы блога):**
- Заголовок: `flex items-center gap-3` + красная черта `w-8 h-0.5 bg-red-500`
- Quote: `bg-red-50 rounded-xl border border-red-100` + SVG кавычки
- Список: `ri-checkbox-circle-fill text-red-600` + двухколоночный grid
- Ссылки: `text-red-600 underline underline-offset-2`
- CTA-карточка: `bg-[#1a1f2c] rounded-2xl text-white` + иконка в красном круге

**Контент — правила:**
- H2 содержит коммерческий запрос («купить ... от производителя»)
- Quote-блок — конкретный тезис о линиях (автоматизация, география, результат). **Не про отдельный станок!**
- Список — состав оборудования линии (из `linesData.ts → composition`)
- Абзац — услуги «под ключ» с 4 внутренними ссылками на `/services/*`
- Ширина: `max-w-7xl` — как все остальные секции

## Шаг 7. FAQ-блок

**Компонент:** `{LineType}FAQ.tsx` в `src/components/lines/`

**Количество вопросов:** 5

**Правило подбора вопросов:** каждый вопрос заточен под конкретный поисковый запрос с просадкой или без покрытия.

**Обязательные темы:**
1. **Цена** — «Сколько стоит линия ...?» (покрывает запросы «... цена / купить / стоимость»)
2. **Сырьё** — «Какое сырьё подходит?» (покрывает «из опилок / из щепы / древесных»)
3. **Запуск бизнеса** — «Что нужно для запуска ... завода?» (покрывает «... завод»)
4. **Производительность** — «Какая производительность?» (покрывает «... линия / оборудование»)
5. **Сравнение/отличие** — «Чем отличается от ...?» (покрывает смежные направления)

**Ответы:**
- Конкретные цифры (цены, площади, мощности, сроки) — из `linesData.ts`
- 1–2 внутренние ссылки на релевантные страницы
- `aText` (plain text для JSON-LD) + `a` (JSX с ссылками для рендера)

**Разметка:** `FAQPage` JSON-LD внутри компонента.

**Расположение на странице:** после SEO-текстового блока, перед Projects.

---

## Шаг 8. Цена в контенте

**Правило:** слово «цена» или числовое значение стоимости должно присутствовать минимум в 3 местах:

1. **Description** — «от X млн ₽»
2. **Подзаголовок секции карточек** — «Цена от X млн ₽»
3. **FAQ** — развёрнутый ответ с ценами всех линий
4. **JSON-LD** — `offers.price` для каждого Product

## Чеклист перед деплоем

- [ ] Title: до 54 символов, главный запрос в начале
- [ ] H1: отличается от title, содержит основной + второй кластер
- [ ] Description: до 160 символов, с ценой и производительностью
- [ ] H2 секции: каждый покрывает отдельный поисковый кластер
- [ ] JSON-LD BreadcrumbList: добавлен в page.tsx
- [ ] JSON-LD ItemList + Product: все линии с ценами
- [ ] JSON-LD FAQPage: внутри компонента FAQ
- [ ] SEO-текстовый блок: двухколоночный, с quote, списком, ссылками
- [ ] FAQ: 5 вопросов, каждый под конкретный запрос
- [ ] Цена: присутствует в description, контенте, FAQ, JSON-LD
- [ ] Внутренние ссылки: минимум 4 на услуги + ссылки на смежные линии
- [ ] `npm run build` — сборка без ошибок
