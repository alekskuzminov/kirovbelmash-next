# Avito Pro Workspace

Рабочая зона по направлению `Avito Pro` для проекта КировБелМаш.

## Цель

Системно готовить и хранить:

- семантические кластеры для объявлений;
- черновики и финальные версии объявлений;
- правила упаковки объявлений под Avito Pro;
- журнал решений и накопленные выводы.

## Текущий фокус

Текущие рабочие группы:

- `оборудование для топливных брикетов`
- `оборудование для производства брикетов`
- `оборудование для производства топливных брикетов`
- `оборудование для брикетов из опилок`
- коммерческие вариации вокруг `цена`, `купить`, `технология`, `комплект`.
- `оборудование для топливных пеллет`
- `оборудование для производства пеллет из опилок`
- `оборудование для изготовления пеллет`
- `оборудование для гранулирования опилок`
- маски `линия гранулирования` и `линия пеллетирования`.

## Принципы работы

1. Одно объявление = один плотный коммерческий интент.
2. Пишем под смысл группы, а не под механический набор точных повторов.
3. Главный ключ выносим в заголовок и первые строки описания.
4. Коммерческие модификаторы вроде `цена`, `купить`, `под ключ`, `за комплект` раскрываем в отдельных смысловых блоках.
5. Все цифры и обещания берём только из текущих материалов проекта или помечаем как гипотезу.

## Стандарт папки объявления

Для каждого объявления создаём отдельную директорию по шаблону:

- `ads/<дата>-<slug>/ad.md` — текст объявления, заметки, характеристики, CTA;
- `ads/<дата>-<slug>/images/` — изображения для карточки, обложки, галереи, схем;
- `ads/<дата>-<slug>/video/` — видео для объявления, если используется.

## Правило обновления

После каждого заметного шага:

- обновляем `WORKLOG.md`;
- сохраняем новое объявление отдельной папкой по дате и теме;
- фиксируем, какие факты были использованы и из каких материалов проекта они взяты.
